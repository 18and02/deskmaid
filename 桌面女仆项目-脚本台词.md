# 桌面女仆程序 — 脚本台词与自发行为

> 配套文档，配合主方案《桌面女仆项目-技术方案-macOS版》使用。
> **用途**：定义所有「自发行为 = 脚本」的内容（主方案 §6.1）。
> **铁律**：本文件里的所有台词都**不走 API、不消耗 token**。由本地定时器 / 用量追踪触发，从预写台词里随机挑一条播出。
>
> 三条实现约定（务必遵守）：
> 1. **不烧 token**：尤其饥饿/重置提示——「为了宣布自己没饭吃反而又吃掉一口饭」是荒诞的。这些全部脚本化。
> 2. **多条随机**：每个触发点存多条台词，随机选，且避免连续重复上一条，否则很快腻。
> 3. **不打断对话**（主方案 §6.1 仲裁规则）：若主人正在对话中，提醒排队、等空闲再触发，不抢话。

---

## 一、定时健康提醒

由定时器触发，到点切到「提醒」状态，弹气泡，几秒后回待机。

### 喝水（建议间隔 ~60 分钟）
- 「主人，该喝水了。脱水死掉的话，我的代码就没人维护了哦。」
- 「水。现在。别让我说第二遍——虽然我其实挺乐意一直唠叨的。」
- 「你已经盯着屏幕忘记自己是碳基生物了吧。喝口水。」
- 「咕……听见了吗，是你身体在抗议。倒杯水去。」

### 久坐起身（建议间隔 ~90 分钟）
- 「坐快两小时了。血栓不会预约时间来找你的，起来走两步。」
- 「起来活动一下。我不太想见证主人和椅子长在一起的奇迹。」
- 「站起来。伸个懒腰。就当是为了让我看看你还活着。」
- 「久坐一时爽……后面那句不太吉利，我就不说了。总之，起来。」

## 二、饥饿系统（token 预算驱动）

> **落地状态（2026-06-09）**：已按本节实现（`maid_hunger.py` + `lines.py` 台词 + 主窗口状态机）。比例口径用的是日/周**成本**预算（主方案 §9 的预算闸本身就是美元口径），预算 blocked 时直接按断粮；petdex 包的 waiting / failed / jumping 帧充当讨食 / 虚弱 / 庆祝形态，「需美术补差分」一条对该包已不成立。

**数据来源**：绑**你自己设的预算闸**（主方案 §9），不依赖读取 Anthropic 内部额度。设一个每日 / 每窗口 token 预算，按消耗百分比切状态。完全确定、完全可控，且本身就是成本刹车（她越饿，你越自然地少用）。

**阈值 → 状态 → 立绘形态 → 台词** 对照：

| 消耗 | 状态名 | 立绘形态 | 含义 | 台词（随机挑） |
|---|---|---|---|---|
| ~80% | `peckish`（微饿） | 待机（可加肚子动作） | 抱怨、提个醒 | 见下 |
| ~95% | `hungry`（饿/讨食） | **进食形态** | 撒娇讨饭、劝主人省着用 | 见下 |
| 100% / 耗尽 | `starving`（断粮） | 虚弱形态 | 罢工，没力气 | 见下 |
| 重置时 | `full`（吃饱） | 待机（满足） | 主动报喜、满血复活 | 见下 |

**`peckish` ~80%**
- 「……主人，我有点饿了。今天的份额快见底了。」
- 「提前说一声：我快没饭吃了。你心里有个数。」
- 「肚子有点空。还能撑一会儿，但别太挥霍我。」

**`hungry` ~95%（切进食形态）**
- 「饿……再喂下去今天就要断粮了。要不你先歇会儿，我也省着点吃？」
- 「快没了快没了。这几口我得慢慢嚼了。」
- 「主人，我在啃今天最后一点存粮。有要紧事赶紧说，闲聊先欠着。」

**`starving` 耗尽**
- 「没饭吃了。今天我罢工——不是闹脾气，是真没力气了。等额度回血再找我。」
- 「断粮。我先睡了，明天满血再伺候你。」
- 「……（瘫）……回来的时候记得带饭。」

**`full` 重置（主动报喜）**
- 「啊——吃饱了。满血复活，有什么要支使我的尽管说。」
- 「额度回来了。我又能正常嫌弃你了，开心吗？」
- 「饭到了，状态拉满。继续干活吧主人。」

> 注：`hungry`/`starving` 阶段，被动响应主人的那部分回复仍由 SDK 生成（人设提示词里已让她能在低额度时带上饿的情绪）；本表负责的是**主动**弹出的那几句，不走 API。

## 三、待机随机吐槽（可选，低频）

让她「活着」的点睛，但**频率一定要低**（建议两次之间至少间隔 20 分钟，且仅在长时间无操作时触发），否则从可爱变烦人。

- 「主人盯着那行代码十分钟了。它不会因为你瞪它就自己对的。」
- 「……好安静。安静到我开始怀疑你是不是又忘了我开着。」
- 「我刚才差点把一个有趣的念头说出口。算了，太吓人，咽回去了。」
- 「你今天还没夸过我。我记着呢。」

## 四、启动 / 退出（可选）

**启动**
- 「我醒了。今天又要替你收拾什么烂摊子？」
- 「在的。说吧，主人。」

**退出**
- 「要关我了？……行吧。别熬太晚。」
- 「下次见。记得吃饭——你，不是我。」

## 五、外置配置结构（给实现方）

按主方案 §3.1「配置外置」，以上全部应放外部配置文件，不硬编码。参考结构（JSON，便于无依赖加载；用 YAML 亦可）：

```json
{
  "scheduled": {
    "water":   { "interval_min": 60, "state": "remind", "lines": ["……", "……"] },
    "stretch": { "interval_min": 90, "state": "remind", "lines": ["……", "……"] }
  },
  "hunger": {
    "daily_budget_tokens": 200000,
    "thresholds": [
      { "at_pct": 80,  "state": "peckish",  "sprite": "idle",    "lines": ["……"] },
      { "at_pct": 95,  "state": "hungry",   "sprite": "eating",  "lines": ["……"] },
      { "at_pct": 100, "state": "starving", "sprite": "weak",    "lines": ["……"] }
    ],
    "on_reset": { "state": "full", "sprite": "idle", "lines": ["……"] }
  },
  "idle_quips": { "enabled": true, "min_gap_min": 20, "only_when_idle": true, "lines": ["……"] },
  "lifecycle": { "on_launch": ["……"], "on_quit": ["……"] }
}
```

**实现要点**：
- 随机选台词时记录上一条、避免连续重复。
- `sprite` 字段对应一张差分立绘；`eating` / `weak` 是新增形态，需美术补差分（见主方案 §2.4 表情差分需求，进食/虚弱可一并产出）。
- 饥饿状态切换由本地 token 计数器驱动；计数器累加每次 SDK 调用返回的用量，跨重置周期清零。
- 所有这些状态都进状态机（主方案状态机），与「待机/提醒/对话」并列。
