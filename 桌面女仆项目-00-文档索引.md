## 当前开发命令入口

仓库根目录提供了一条统一的日常回归命令：

```bash
./dev_checks.py
```

- 默认跑 `quick`：权限自检 + 桌面输入/桥接回归。
- `./dev_checks.py list` 可查看所有 profile。
- `./dev_checks.py packaged` / `./dev_checks.py bundle` 可构建 `.app` 并从 bundle 内跑健康验证。
- `./dev_checks.py signed` 可构建 `.app`、执行 codesign，并校验签名；需要先准备 `DESKMAID_CODESIGN_IDENTITY`。
- `./dev_checks.py notarize` / `./dev_checks.py release` 可继续走 notarization / staple / Gatekeeper；推荐先准备 `DESKMAID_NOTARY_KEYCHAIN_PROFILE`。
- `./dev_checks.py tcc` / `./dev_checks.py realmachine` 可继续做打包态 Launch Services + 实机 TCC 清单回归。
- 只想先确认这台机器有没有签名证书时，可直接跑 `.venv/bin/python -u build_macos_app.py --list-signing-identities`。
- 只想单独查打包/TCC 时，可直接跑 `.venv/bin/python -u build_macos_app.py --verify-health`。
- 第一次配置发布机时，直接看主方案 §10.1「真机签名 / 公证傻瓜式清单」。
- 更完整的专项/全量命令见主方案新增的「当前开发 / 回归命令」小节。
