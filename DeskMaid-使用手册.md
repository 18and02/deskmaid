# DeskMaid 使用手册

更新日期：2026-05-29

## 1. 这是什么

`DeskMaid` 是一个常驻在 macOS 桌面的女仆助手。

当前发行包已经包含：

- `Deskmaid.app`
- 桌面透明角色壳
- 对话与工具调用运行时
- 本地长期记忆、预算和隐私边界
- Calendar / Reminders / Mail / 桌面桥接能力

你**不需要额外安装 Python 或 Claude CLI**。  
你只需要准备一个可用的 **Claude API key**。

## 2. 安装与启动

1. 下载 `DeskMaid.dmg` 或 `DeskMaid.zip`
2. 如果是 `DeskMaid.dmg`，打开后把 `Deskmaid.app` 拖到 `Applications`
3. 如果是 `DeskMaid.zip`，先解压，再把 `Deskmaid.app` 拖到 `Applications`
4. 双击启动

如果 macOS 提示应用无法打开：

1. 在 Finder 里右键 `Deskmaid.app`
2. 选择“打开”
3. 再点一次“打开”

如果系统仍拦截：

1. 打开“系统设置 -> 隐私与安全性”
2. 在页面底部找到被拦截提示
3. 点“仍要打开”

说明：如果这份发行包还没做 Apple 签名 / 公证，第一次启动时出现这类提示是正常的。

## 3. 首次启动

第一次启动会进入 Setup / 首启设置。

建议按这个顺序填：

1. `Claude API key`
2. 你的称呼
3. 回复语言
4. 界面语言
5. 预算档位
6. 数据边界确认
7. 是否自动隐藏

保存后，女仆就能开始真正对话。

如果先不填 API key，应用可以打开，但只能显示外壳，不能正常聊天。

## 4. 日常使用

### 输入

- 点击女仆身体可打开输入入口
- 输入框支持**多行输入**
- 支持**拖入附件**

### 右键菜单

右键女仆可打开常用入口，例如：

- Setup
- Permission Health
- Privacy Boundary
- Budget Status
- Long-term Memory
- Auto DND
- Quit

### 工具调用

当她要操作日历、提醒事项、邮件或桌面时，会先弹权限确认。

你可以：

- 只允许这一次
- 记住到本次会话

高风险动作，比如发送邮件，仍然会保留人工确认。

## 5. 需要哪些系统权限

根据你实际使用的功能，DeskMaid 可能会请求这些权限：

- **辅助功能**：窗口聚焦、按键、粘贴等桌面输入能力
- **自动化**：控制 Calendar / Reminders / Mail / System Events

如果权限没开，可以在应用里打开：

- `Permission Health`

它会告诉你缺哪一项，并提供直接跳到系统设置的入口。

## 6. 为什么自动化页里有时看不到 Deskmaid

这是 macOS 的权限显示方式和签名状态共同导致的，不一定是应用坏了。

系统会把自动化权限记在“发送控制请求的进程”名下。  
因此在“系统设置 -> 隐私与安全性 -> 自动化”里，你有时看到的可能是：

- `Deskmaid`
- `osascript`
- 其他宿主进程名

如果某个功能明明触发过，但没看到 `Deskmaid`，也顺手看看是否记在 `osascript` 或当前宿主应用那一项下面。

另外，如果当前这份包还是 **ad-hoc / 未做正式 Apple 证书签名**，自动化页里的应用名也可能不稳定。
这时更常见的表现就是权限条目挂在 `Terminal`、`Codex`、`claude` 或 `osascript` 名下，而不是稳定显示成 `Deskmaid`。

你可以在应用里的 `Permission Health` 看“打包后的签名身份”这一项：

- 如果那里提示 `ad-hoc`，说明这不是你的操作问题，而是当前发行包还没进正式签名 / 公证链
- 如果已经是正式签名，再从 `/Applications/Deskmaid.app` 直接触发一次 Calendar / Reminders / Mail，系统记录通常会更稳定

## 7. 主要能力

当前版本已经可以做这些事：

- 日常聊天
- AskUserQuestion 追问弹窗
- 会话续接
- 长期记忆查看、新增、编辑、删除
- 隐私边界解释与快捷改写
- 预算状态查看
- 自动免打扰总开关
- Calendar 读取与写入
- Reminders 读取与写入
- Mail 读信、建草稿、附件、人工确认发送
- 桌面桥接：打开 App、打开 URL、窗口聚焦、剪贴板、粘贴文字、按键

## 8. 常见问题

### 8.1 打开了但不回复

先检查这几项：

1. Setup 里是否已经填好 `Claude API key`
2. 网络是否正常
3. `Permission Health` 里是否有明显报错
4. 预算是否已经被硬闸拦住

### 8.2 日历 / 提醒事项 / 邮件不能用

先打开：

- `Permission Health`

然后按提示补：

- 自动化
- 辅助功能

补完后回到应用里点一次“刷新”。

### 8.3 发送邮件前为什么还要我确认

这是有意保留的人机确认链路。  
发送类动作属于高风险操作，当前版本不会默认静默放行。

### 8.4 隐私边界把我的输入拦下了

这是正常护栏。

你可以：

- 直接用输入框里的快捷改写按钮
- 改成更少敏感信息的表述
- 改成“仅本机处理”的方式

### 8.5 自动免打扰为什么没生效

先确认：

1. 自动免打扰检测总开关是否已打开
2. 当前场景是否属于已支持检测的范围
3. 是否手动关闭过自动隐藏

当前版本已支持全屏、会议/通话、共享/录屏、摄像头占用和系统 Focus 的场景检测，但不是所有系统状态都百分百覆盖。摄像头占用只会静音提醒，不会自动隐藏立绘（隐藏只跟共享/录屏信号走）。

## 9. 数据与隐私

当前版本的数据边界大致如下：

- 本机保存：偏好设置、预算账本、长期记忆、本地状态
- 发往模型侧：你明确发起的一轮对话内容，以及被允许送出的必要工具结果

已经接上的保护包括：

- 常见敏感信息脱敏
- 高敏内容拒传
- 长期记忆本地加密
- 预算硬闸
- 工具权限确认

## 10. 卸载

如果不再使用：

1. 退出 `Deskmaid.app`
2. 删除应用本体
3. 如需彻底清理，再删除本机保存的状态文件和私有数据

是否保留本地记忆 / API key / 偏好，取决于你是否同时清理这些本地文件。
