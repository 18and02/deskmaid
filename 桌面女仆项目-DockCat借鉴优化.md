## 0. 先划清一条根本差异(决定哪些能借、哪些不能)

DockCat 和 deskmaid 外形几乎一样:常驻、透明异形窗、桌宠、会提醒、可拖动、支持自定义形象。但有一条根本分叉:

| | DockCat | deskmaid |
|---|---|---|
| 脑 | **没有脑**——纯本地脚本行为,完全不联网、不传数据 | Claude Agent SDK(云端),对话 / 记忆 / 操作 Mac 全靠它 |
| 卖点 | 安静陪伴 + 「出门收集」轻玩法 | 能对话、能动手(Mail / Calendar / Reminders)的腹黑助理 |
| 自发行为 | 全部 | 只占 §6.1 那一半(脚本管定时) |

**结论**:DockCat 在「**壳 + 自发行为 + 资源系统 + 轻玩法**」这半边打磨得很完整,正好对应 deskmaid 的**第 1 层(壳)**和 **§6.1(脚本自发行为)**——这半边可以大胆借。而「脑」那半边,deskmaid(§6/§7/§8 + §14 隐私 + §16 免打扰 + §9 预算)已经远比 DockCat 强,不用借。

换个说法:**deskmaid 的「降级态」目标,本质就是退化成一只 DockCat**(见 §5)。所以 DockCat 是 deskmaid 离线那一半的天然参考实现。

## 2. P0 — 资源系统 manifest 化(A + B + C + G 一起做)

### 现状(2026-05-29 校正)
这一块的**主链路已经吃到第一轮红利**:deskmaid 现在已经按 `Maid/assets/packs/<pack-id>/manifest.json` 组织资源包,pack 信息面板与缺失状态 fallback 也已接上,不再依赖早期那套手改 `SPRITE_SETS` 的落位方式。

因此,本节现在更适合作为「**接下来还能怎么把资源系统做得更完整**」的借鉴清单,而不是“从零开始把硬编码改掉”的施工单。

### DockCat 的解法(可直接照搬结构,不照搬猫的状态语义)
资源包 = 一个文件夹,含 `manifest.json` + 按状态分类的透明 PNG:

```
Maid/assets/packs/<packname>/
  manifest.json
  poses/
    idle/      *.png      ← 同一状态可放多张,进入时随机抽
    blink/     *.png
    alert/     *.png
    excited/   *.png
    enter/     *.png
    exit/      *.png
    sleepy/    *.png
    peckish/   *.png
    held/      *.png       ← 顺便补 E
  animations/
    walk/      walk_01..NN.png   ← 预留,将来要走动时按序循环
```

`manifest.json`(字段抄 DockCat,但 `poses` 用 deskmaid 自己的 **8(+held) 态**,不要照搬猫的 resting/dialogue/transition/held 四态):

```json
{
  "id": "gemini-maid",
  "name": "梁吉吉",
  "author": "...",
  "canvas_width": 480,
  "canvas_height": 480,
  "default_anchor": { "x": 0.5, "y": 0.92 },
  "poses": {
    "idle": "poses/idle", "blink": "poses/blink",
    "alert": "poses/alert", "excited": "poses/excited",
    "enter": "poses/enter", "exit": "poses/exit",
    "sleepy": "poses/sleepy", "peckish": "poses/peckish",
    "held": "poses/held"
  },
  "animations": { "walk": { "fps": 3, "frames": [] } }
}
```

### 资源包下一步还值得补的点
1. **同一 pose 的多图随机**(B):`idle` / `alert` 这类状态若能支持 3-4 张细微差异图,待机观感会比单张定格自然很多。
2. **walk 序列动画预留吃满**:manifest 已经适合继续扩 `animations/walk/`;等成品资源到位后,可按文件名顺序循环,把“出门 / 闲时溜达”做得更活。
3. **fallback 层级继续做细**(G):当前基础 fallback 已有,后续仍可继续补“缺姿势 → 默认包同姿势 → 默认 idle”这类更稳的层级回退。
4. **pack 元数据继续做厚**:作者、版本、说明、预览图、支持状态列表、缺失提示等信息,都值得比当前基础版再完整一些。
5. 当前 `_current_state_key()` 的优先级逻辑依然**不用大改**——后续增强仍主要发生在「state key → 具体资源」这一层,壳的状态机可以继续保持稳定。

### 收益
- 加表情 / 换装 / 做第二只角色 = **丢文件夹,零改代码**;真正兑现 §4 配置外置。
- 为 P1「出门收藏品」和将来「走路动画」铺好同一套数据驱动地基。
- 多图随机让待机不再死板(成本几乎为零)。

### C:把 DockCat 的美术一致性规则直接抄进 §3.1「替换后必做校验」
DockCat 自定义指引里这几条,deskmaid 该原样纳入规范(其中 deskmaid 已部分做到的标 ✓):

- ✓ 透明背景 PNG;✓ 所有状态同画布尺寸(当前已归一到 480)。
- ⚠ **脚底 / 身体底部基线在所有图里对齐**——现在我们是「居中 pad 到 480 画布」,脚底没对齐,**切状态时会上下微跳**;应改成「按脚底基线对齐后再 pad」。
- ⚠ **禁止地板 / 投影 / 场景 / 文字 / 无关道具**——当前 Gemini 套底部还留着青色脚印阴影,该擦掉(否则呼吸/切态时阴影跟着动很穿帮)。
- ⭘ **左右自动镜像**:只画一个朝向,代码翻转出另一个朝向——将来要「朝鼠标」或走动时省一半美术,现在静态可不急。
- ✓ 统一画风(线宽 / 配色 / 光照)。注意 `maid.png`(emote 用的原画处理版)笔触明显比 Gemini 套松,作为「突然来劲」的 emote 反而是特性;但若要当常驻状态会风格跳。

## 3. P1 — 「出门」玩法循环(D):DockCat 最该借的特色

deskmaid 现在已经把这条线的**基础版**接上了:本地见闻 / 收藏品双轨、回执卡片与状态存档都已经能工作。所以本节现在讨论的重点不再是“要不要做”,而是“**怎样把这条轻玩法做得更像正式系统**”。

DockCat 的「出门」仍然是极好的参考:它把“低打扰陪伴”“轻收集”“状态切换”揉成了一个非常自然的循环。对 deskmaid 来说,这条线仍然最有价值的地方,是它和预算 / 降级 / 饥饿 / 免打扰这些现有机制天生能咬合。

对 deskmaid 来说,这个机制的价值不只是「好玩」,而是它**天然耦合了 deskmaid 已有的四个系统**——这才是它值得借的原因:

### 3.1 跟现有机制的耦合点(贴合的核心)
| 耦合 | 怎么接 |
|---|---|
| **§9 预算 / 闲时降频** | 出门 = 主动进入低耗窗口。她出门期间**不发起 API 调用** → 直接帮省 Agent SDK 额度。§9 现有的「闲时自动切保守预算档」可以**叙事化成「她出门遛弯去了」**,比一个看不见的后台降档更有感知。 |
| **§15 降级 / 断线** | 断网或预算见底时,§15 现在的设计是回一句脚本降级台词「我连不上脑子」。**更好的包装:让她"出门"**——把故障态叙事成主动行为,体验从「坏了」变成「她去忙了」。§15 本就说「断/饿各配一张差分(断线/打盹形态)」,**出门态正是这张差分的积极版**。 |
| **饥饿系统(脚本台词文档)** | 出门可「带回吃的」补饥饿值,形成轻循环:饿 → 出门 → 带回收藏品/食物 → 饱。给常驻一个低频正反馈。 |
| **§16 免打扰** | 共享屏幕 / 开会 / 录屏时,与其硬「隐藏立绘」,可叙事成「她识趣地出门回避了」,场景结束再回来。比凭空消失自然。 |

### 3.2 见闻 event 的双轨(复用 §15 在线/离线分轨)
- **离线 / 省钱轨**:预置文案,抄 DockCat 的 `events.json` 结构(`id` / `type` / 中英文案),放 `Maid/assets/outing/events.json`。断网、没预算、降级时用这套——**绝不耗 API**。
- **在线轨**:有预算且在线时,让 **Claude 用人设口吻现编**一条腹黑见闻(「我在你的下载文件夹门口踩到一个三个月没动的安装包,帮你默哀了一下」)。这比静态文案活,且正好喂给 §6 的 SDK + §7 记忆(出门见闻可以引用记忆里关于主人的事实)。

### 3.3 收藏品 collectable(纯本地数据驱动)
- 抄 DockCat `collectables.json`:`id` / 名 / `rarity` / 图路径,放 `Maid/assets/outing/collectables/`。
- 落进 **§9 已有的本地账本**(它已经在记账单/日/周成本,加一张「收藏品清单」表是顺手的)。
- 隐私零负担——纯本地,不上云,符合 §14。

### 3.4 状态机改动(很小)
- 新增 `OUTING` 状态(立绘:空场景 / 背影 / 一扇门,或干脆透明只留气泡)。
- 触发:① 脚本定时(§6.1 自发行为,不耗 API);② 主人主动「你去忙会儿」;③ §15 降级时自动;④ §16 高敏场景时自动。
- 回来:切到 `dialogue`/`alert` 报告见闻 → 回 `idle`。`enter` / `exit` 过渡帧的基建可直接复用。

## 4. P2 — 锦上添花

### E. `held` 被抱起姿势
- 加进 §3.1 状态集(变 9 态)。
- **逻辑现成**:第 1 层已有 `_dragging` 标志,拖动时切 `held` sprite 即可,优先级放在 emote 之上、alert 之下。
- 缺的是美术:一张「被拎起来 / 腿悬空 / 略惊讶」差分(红眼签名保留)。

### F. 设置粒度对照
DockCat 的设置清单可作为 deskmaid 设置项的对照表。deskmaid 首启已覆盖 API key / 命名 / 预算 / 数据边界,可继续补 DockCat 有而我们没单列的:
- **对主人的称呼**(现在 system prompt 写死「主人」,该外置成可改)。
- **显示缩放**(对应现有的 `--sprite-dpr`,该升级成持久设置而非 CLI flag)。
- **各状态时长 / 提醒间隔 + 自定义文案**(现在是常量 / CLI,该落进设置文件)。
- 全部归 §4 配置外置 + §12 首启之后的设置面板。

## 5. 明确不借 / 分叉(H)

- **完全本地、零联网**:不可借——deskmaid 的命脉就是云脑(§0 / §6)。但 DockCat「云脑不在时纯本地脚本行为照常跑」的**框架**正是 §15 降级态要的:断网 / 没预算时,微动 + 提醒 + 出门这些本地自发行为照常,只是不能「思考」。**建议把这个类比直接写进 §15:降级态 = 临时退化成一只 DockCat。**
- **住在 Dock 上沿 + 动态 Dock 图标(`icon_sleep` / `icon_empty`)**:deskmaid 是 `LSUIElement` 附件型、可浮在任意位置(§5.5),不绑 Dock,这条不适用。
- **PolyForm Noncommercial 许可**:那是 DockCat 自己的授权,且是编译产物,**不能拷代码**;我们只借「设计模式 / 数据结构 / 资源约定」这类思路,不碰它的实现。

## 6. 建议落地顺序 + 该回填到主方案哪几节

### 落地顺序(按 2026-05-29 真实进度重排)
1. **C — 美术一致性规则**(纯规范,零代码):把 DockCat 那几条继续写实到 §3.1「替换后必做校验」里,优先解决脚底基线、阴影/地板残留、成品资源 QA。
2. **B + G 深化 — 资源包体验补完**:manifest / pack info / fallback 已落地,下一步更值的是多图随机、更完整 pack 元数据、以及更稳的层级回退。
3. **D 深化 — 出门玩法从基础版走向正式版**:继续补在线见闻、回执卡片 polish,以及和预算 / 降级 / 免打扰更紧的叙事绑定。
4. **E / F**(随手补):`held` 姿势、称呼/缩放/时长/文案外置。

### 回填主方案的位置(本文是提案,真正定稿请并进①)
- **§3.1**:manifest 化步骤已经并进主方案;现在更该继续补的是 DockCat 那套美术一致性 checklist、`held` / `walk` 资源约定、以及替换后 QA 规则。
- **§4 配置外置**:继续做厚「资源包 manifest / pack 元数据 / 用户设置项(称呼 / 缩放 / 时长 / 文案)」。
- **§6.1 自发行为**:把“出门”从基础闭环继续推进成更完整的本地轻玩法。
- **§9 预算**:继续把「闲时降频 / 省预算 / 预算见底 / 重置回来」这些状态叙事化成出门行为。
- **§15 降级**:继续加强「降级态 = 退化成 DockCat」这层包装,让断线/预算见底比纯报错更自然。
- **§17 Backlog**:主方案的 B9 已经变成“资源包 / 出门态继续打磨”;本文剩余更像是它的细化子项,而不是另起一组新编号。

### 一句话总结
DockCat 教给 deskmaid 的不是「脑」,而是**「壳怎么做得更耐看、更可换装、更有轻玩法」**:把硬编码的精灵集升级成数据驱动的资源包(P0),再把「出门收集」这个轻循环接到 deskmaid 已有的预算 / 降级 / 饥饿 / 免打扰系统上(P1)——后者几乎是免费的体验增量,因为底层机制 deskmaid 都已经有了。

## 7. 复看更新(对照《技术方案-macOS版_最新版》,2026-05-29)

把《_最新版》通读了一遍。和上次最大的不同是:本文里提的几条大方向,现在已经有一部分**不再停留在建议**,而是进入了“已落地基础版 + 仍可继续打磨”的状态。最近的增量除了脑侧继续变细,壳侧也已经明显前进:
- §6.2:权限弹窗现在显示工具**风险档** + 本档剩余次数 + 整轮剩余额度 + 「本会话始终允许」;`permission_request` trace 同字段。
- §7:长期记忆命中后,先在本地把 `like/dislike` 偏好**合并**,再按预算档**裁剪**后才上云。
- §9:工具调用按 `low / medium / high / critical` 做**单轮分层配额**;记忆命中走 `标准 / 轻量 / 省预算` 三档。
- §11 / §12:资源包已经 manifest 化,pack 信息 / fallback 已接上;“出门态”也已有本地见闻 / 收藏品双轨基础版与回执卡片。

所以现在更准确的说法是:DockCat 在「脑」侧依然帮不上,但它在「壳 / 资源系统 / 轻玩法」上的借鉴,已经有一部分被 deskmaid 吃进主线了。

> **结论(2026-05-29 校正):** 本文里最硬的两条借鉴已经不再是“建议”而是“已落地主链路”:**P0 的 manifest / pack info / fallback**,以及 **P1 的出门态见闻 / 收藏品双轨基础版**。本文剩余最高价值的部分,变成了**继续 polish 的方向**:多图随机 / walk / held / pack 元数据 / 更深的预算与降级叙事绑定。

更关键的是:新版的几处推进,反而让某些借鉴**更刚需**了。

### 7.1 §9 预算变细 → 「出门」叙事更刚需(强化 P1)
新版 §9 现在有四种预算状态——**在线充裕 / 闲时保守档 / 省预算档 / 当期见底直接拦**,外加「预算窗跨天跨周重置提示」。而 deskmaid 已经有了“出门态”这层基础壳,所以下一步最值的是把这些预算状态**更明确地映射**到出门叙事上:

| §9 预算状态 | 现在的表现 | 用「出门」叙事化后 |
|---|---|---|
| 闲时保守档 | 后台悄悄降档 | 「我去外面溜达会儿」——低调出门,顺便省额度 |
| 当期预算见底 | 发请求前直接拦 | 「这个月额度被你榨干了,我出门躲债,等回血」——比冷冰冰「预算不足」体验好太多 |
| 预算窗重置 | 一句本地提示 | 「我回来啦」= 额度窗重置归来 |

**一句话:§9 越精细,越需要「出门 / 降级态」这层把不可见的预算状态翻译成人格行为的外壳。** 建议把这层映射明确写进 §9 ↔ §15。

### 7.2 降级态分层(优化:新预算档撑得起更细的降级阶梯)
§15 现在是「饿 / 断」两态。新版 §9 已经把预算切成多档,§15 可顺势升级成一条**四级降级阶梯**,每级配一个语气 / 差分:

1. **在线充裕** → 正常腹黑
2. **在线省档**(闲时 / 预算紧) → 话变少、慵懒(可直接复用 `sleepy`)
3. **预算见底**(本期被拦) → 「出门」态(P1 的积极版,等窗口重置)
4. **断网** → 断线 / 打盹差分(§15 已提)

好处:把「饿(没预算)」从一个布尔升级成「省着花 → 花光出门」的连续叙事,跟 §9 的档位天然对齐,且每一级都能复用已有 sprite 状态。

### 7.3 陪伴统计(DockCat 借鉴新角度,零上云)
DockCat 本地记三类统计:陪伴时长、完成喝水 / 走动提醒次数、出门收藏品。deskmaid 记忆层已经是**本地 + 加密**(§7 / §14),顺手加一层**轻量关系统计**(纯本地、零上云、零隐私成本)完全在调性内:
- 「我们一起熬了 N 个夜了」「这周我提醒你喝水 12 次,你只听了 3 次」——腹黑吐槽的现成素材。
- 这些 stat 偶尔塞进 system prompt 能给对话连续感(「主人最近通宵频率有点高」)。
- 落点:§7 记忆层旁挂一张本地 stats 表(跟 §9 本地账本同级,同样不上云)。

### 7.4 走路动画 = 零 API 的活气(重申 P0 预留的 walk)
新版没动状态集。P0 的 manifest 里已给 `animations/walk/` 预留位。**走动是纯 §6.1 本地自发行为,一个 token 都不花**,却显著提升「活着」感——尤其配合 7.1 的出门 / 省档叙事,「她在桌面边走来走去」比定格干站自然得多。建议成品美术阶段连 walk 4 帧一起出(DockCat 的横向 4 帧条带 + 自动镜像规约可直接照搬)。

### 7.5 档位视觉语言统一(小,设计系统层面)
新版 §6.2 让权限弹窗显示**风险档**(low/medium/high/critical),DockCat 收藏品有 **rarity 档**,§9 还有**预算档**——三者本质都是「分级徽标」。建议统一一套**颜色 chip 语汇**(如 critical=红 / high=橙 / medium=黄 / low=灰),让风险档、稀有度、预算档共用,省得每处各自发明配色。

### 7.6 纯优化:文档正在「一文两用」(建议拆 CHANGELOG)
这条与 DockCat 无关,是读 46KB 全文的观感:**①现在同时是「权威设计 / 决策记录」+「逐节进度日志」**——几乎每节都挂了一长段「当前已落地…」。随着进度堆积,①越来越难一眼读出「设计意图」。建议把「进度 / 已落地」抽成单独的 `进度同步 / CHANGELOG`,让①回归干净的权威设计;这也更符合索引里「①是唯一权威」的定位——权威文件越短越稳。

### 7.7 当前最高杠杆的剩余项,仍有一块正好是 DockCat 帮得上的
§12 状态表里,资源包 manifest、出门态基础版、预算/隐私/免打扰这些主链路已经大都进了 ✅ / 🟡。所以现在更准确地说,**空白不再是“有没有资源系统/出门态”,而是“这套壳能不能做得更成品”**——这恰好仍对应本文 **C(美术一致性) + B(多图随机/walk) + E(`held`) + D 深化(更正式的出门系统)**。

新版不是“壳这边原地未动”了,而是已经把 manifest 资源包和出门态基础版接进主线。**因此本文现在最有价值的,是继续告诉你这套壳下一步该怎么从“能用”做成“更耐看、更完整、更有轻玩法感”。** 优先级也随之重排:先把资源包体验和美术一致性做扎实,再把出门 / 降级 / 预算这条叙事链做深。
